Disadvantages of notion.so

Last update @2019/05/05

While I am trying to use this platform, lets compile list of disadvantages. This page is a work in progress and will be updated.

  • Missing reflection of the system itself. For example, page with list of all templates.
  • Missing (optional) edit bars for all kind of text (bold, italic, whatever).
    • I know that they are dynamically shown when you select piece of text, but I would like to see them in toolbar.
  • /commands are really annoying in text, command pallete ala sublime would be imho much better / cleaner in UI.
    • Previous sentence triggered command popup. This makes typing regular / really annoying.
    • Also I hate when I have to type /turnintoquote into middle of my text, then accidentally close the menu and then have to manually delete this slash-command from the text, because it can't be triggered again only by typing (no key shortcut).
    • Also they don't work for selections of multiple lines / elements.
  • Kind of strange way how to integrate data inside.
    • For example, how can I get all date data points creted in todo into the calendar view?
    • Or all todo boxes in some sub-section into one aggregated page?
  • Inability to create multiple workspace views in the left menu bar
    • Workspace view is nice, but I would like to see there several workspaces used for different stuff (team sharing, personal web, whatever).
  • No RSS / Atom feeds.
  • No ability to create own objects.
  • You don't see public icon on public items.
    • Also no list of all public items in case you want to make something accidentally-made-public private.
  • Import of the .csv and .html fucking sucks ass. .csv is just some kind of basic parser and good luck getting the data into table. Tables in HTML are ignored completely.
  • Quotations (<blockquote>) can't contain advanced formatting (lists, checkboxes, whatever).
  • Only two levels of headers.
  • Breadcrumbs are broken for pages (symlinks?) present in multiple sections.
    • For example if you see this public page, breacrumbs at the top of the page doesn't work.
  • You can't have private subpages of public pages. This is really annoying when you want to create some kind of private TODO.
  • @date fucking sux. I really hate how I have to manually change format of the date each time.
    • Also there is no way how to get list of all my @date items? Wtf?
  • Notion has weird bugs under firefox.
    • Cursor keys don't work in paragraphs same way as you expect and how they work in Chromium.
  • No picture editor. At least basic drawing would be appreciated.
  • HTML import only works for one file at a time. This sux ass.
  • You can put sub-page directly to the kanban (board view), but you then can't see the sub-page in the left menu.
  • Missing ability to change fonts / customize technical details.
  • No ability to force logoff of all connected sessions.

Philosophical

  • No scripting
  • No API so far
    • No way to automatically backup your data → sux
  • Close-sourcedness
  • No self-hosting solution
  • Requires periodic payments
  • No ability to integrate itself
    • While it allows to integrate content from (absurdly few) content providers, there is no simpe way how to integrate content created here in other aps.
  • It looks like "databases" created in-app are not really databases. Give me sql admin / API iterator for fucks sake.
  • Fucking continuous-markup mode of editing known from word. I mean you make a word italic and then want to type something else, but you still type in italic. I hate this and I consider the mode used in CherryTree much better.
  • No user defined key shortcuts.
